Get ollim-bot running and send your first message in 7 steps.

## Before you start

You need:

- **A computer** running macOS, Windows, or Linux
- **A Discord account** — [create one here](https://discord.com/register) if you don't have one
- **A Claude subscription** — Pro ($20/month) or Max ($100/month) at [claude.ai](https://claude.ai). Pro works fine to start; see [model providers](/self-hosting/model-providers) for alternatives

<Note>
ollim-bot is a single-user bot designed to run as a personal assistant
in Discord DMs. No server is required.
</Note>

## Open a terminal

You'll paste commands into a terminal throughout this guide. Open one now:

<Tabs>
  <Tab title="macOS">
    Press **Cmd + Space**, type **Terminal**, and press **Enter**.
  </Tab>
  <Tab title="Windows">
    Press the **Windows key**, type **PowerShell**, and click **Windows PowerShell**.
  </Tab>
  <Tab title="Linux">
    Press **Ctrl + Alt + T**.
  </Tab>
</Tabs>

A window with a blinking cursor appears — this is where you'll paste
commands from this guide. To run a command, paste it in and press
**Enter**.

<Warning>
Keep this terminal window open for the entire guide. If you close it,
you'll need to open a new one and navigate back to the right folder.
</Warning>

<Steps>
  <Step title="Install Git and uv">
    ollim-bot needs two tools: **Git** (downloads code) and **uv**
    (manages Python, the language the bot is written in).

    ### Install Git

    <Tabs>
      <Tab title="macOS">
        Git is pre-installed on macOS. Verify by pasting this and pressing **Enter**:

        ```bash
        git --version
        ```

        You should see something like `git version 2.39.0`. If you see
        "command not found", download Git from
        [git-scm.com](https://git-scm.com/) and run the installer.
      </Tab>
      <Tab title="Windows">
        Download the installer from [git-scm.com](https://git-scm.com/)
        and run it. Use all the default options.

        After it finishes, **close PowerShell and open it again** so it
        can find Git. Then verify:

        ```powershell
        git --version
        ```

        You should see something like `git version 2.44.0.windows.1`.
      </Tab>
      <Tab title="Linux">
        ```bash
        sudo apt install git
        ```

        On Fedora, use `sudo dnf install git` instead.
      </Tab>
    </Tabs>

    ### Install uv

    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        curl -LsSf https://astral.sh/uv/install.sh | sh
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
        ```
      </Tab>
    </Tabs>

    After uv installs, **close your terminal and open a new one** (same
    way as before) so your computer can find the new command. Then
    verify:

    ```bash
    uv --version
    ```

    You should see a version number like `uv 0.6.0`.
  </Step>

  <Step title="Download and install ollim-bot">
    Paste each command below one at a time, pressing **Enter** after each
    one.

    Download the bot's code:

    ```bash
    git clone https://github.com/Ollim-AI/ollim-bot.git
    ```

    Move into the downloaded folder:

    ```bash
    cd ollim-bot
    ```

    Install the bot:

    ```bash
    uv tool install --editable .
    ```

    This also installs
    [`claude-history`](https://github.com/Ollim-AI/claude-history), a
    tool that lets the bot search past conversations so it can remember
    things.

    **Close your terminal and open a new one**, then verify the bot
    installed:

    ```bash
    ollim-bot --help
    ```

    You should see a list of commands starting with
    `ollim-bot -- ADHD-friendly Discord assistant`. If you see "command
    not found", see [troubleshooting](/development/troubleshooting).
  </Step>

  <Step title="Create a Discord bot">
    Go to the [Discord Developer Portal](https://discord.com/developers/applications)
    and click **New Application**. Give it any name (e.g. "ollim-bot").

    <Frame>
      <img src="/images/discord-new-app.png" alt="Discord Developer Portal with a red arrow pointing to the New Application button in the top right" />
    </Frame>

    In the left sidebar, click **Bot**. Then click **Reset Token** (or
    **Add Bot** if you see that instead) and copy the token that
    appears — save it somewhere safe, you'll need it in the next step.

    <Frame>
      <img src="/images/discord-bot-token.png" alt="Screenshot of the Bot page showing the Reset Token button and the Public Bot toggle below it" />
    </Frame>

    <Warning>The bot token is shown only once. If you lose it, click **Reset Token** again to get a new one.</Warning>

    Scroll down on the same page to **Privileged Gateway Intents** and
    turn on **Message Content Intent** (this lets the bot read your
    messages).

    <Frame>
      <img src="/images/discord-message-intent.png" alt="Screenshot of the Privileged Gateway Intents section with the Message Content Intent toggle switched on" />
    </Frame>

    <Note>
    Need more help? See [Discord bot setup](/self-hosting/discord-bot-setup) for a detailed walkthrough of every setting.
    </Note>
  </Step>

  <Step title="Configure the bot">
    Back in your terminal, navigate to the ollim-bot folder. If you
    closed your terminal since step 2, open a new one first:

    ```bash
    cd ollim-bot
    ```

    Create the configuration file:

    <Tabs>
      <Tab title="Linux / macOS">
        ```bash
        cp .env.example .env
        ```
      </Tab>
      <Tab title="Windows">
        ```powershell
        copy .env.example .env
        ```
      </Tab>
    </Tabs>

    Open the file in a text editor:

    <Tabs>
      <Tab title="macOS">
        ```bash
        open -e .env
        ```

        This opens the file in TextEdit.
      </Tab>
      <Tab title="Windows">
        ```powershell
        notepad .env
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        nano .env
        ```

        Use arrow keys to move around. When done editing, press
        **Ctrl + O** then **Enter** to save, then **Ctrl + X** to exit.
      </Tab>
    </Tabs>

    Find these three lines and fill in the values:

    ```bash title=".env"
    DISCORD_TOKEN=paste-your-bot-token-here
    OLLIM_USER_NAME=YourFirstName
    OLLIM_BOT_NAME=Ollim
    ```

    - Replace `paste-your-bot-token-here` with the token you copied in step 3
    - Replace `YourFirstName` with your name (the bot will call you this)
    - Replace `Ollim` with whatever you want to call the bot

    Save the file and close the editor.
  </Step>

  <Step title="Install to your Discord account">
    Back in the [Discord Developer Portal](https://discord.com/developers/applications),
    click your application, then click **Installation** in the left
    sidebar.

    Under **Installation Contexts**, check **User Install** and uncheck
    **Guild Install**. Set the install link dropdown to **Discord
    Provided Link**. Under **Default Install Settings**, click the
    scopes dropdown and add `applications.commands`. Then click **Save
    Changes** at the bottom.

    <Frame>
      <img src="/images/discord-installation.png" alt="Screenshot of the Installation page with User Install checked, Discord Provided Link selected, and applications.commands scope added" />
    </Frame>

    Copy the **Install Link** shown on this page and open it in your
    browser. Click **Add to my apps** to install the bot to your Discord
    account.

    Finally, go back to **Bot** in the left sidebar and uncheck **Public
    Bot** (the bot is personal — only you need it).

    <Warning>
    You must set the install link and install the app *before* unchecking
    **Public Bot**. Discord does not allow private apps to have an
    install link, so doing these out of order causes an error.
    </Warning>
  </Step>

  <Step title="Start the bot">
    In your terminal, navigate to the ollim-bot folder if you're not
    already there:

    ```bash
    cd ollim-bot
    ```

    Start the bot:

    ```bash
    ollim-bot
    ```

    <Warning>
    Keep this terminal window open. The bot runs as long as the terminal
    is open — closing the terminal stops the bot.
    </Warning>

    If this is your first time, the bot sends you a Discord DM with a
    sign-in link. Open the link and log in with your Anthropic account
    (the one with your Claude subscription). The bot waits for you to
    finish signing in before continuing.

    Once ready, the bot sends you a startup DM in Discord.

    <Tip>
    Bot not starting? See [common startup issues](/development/troubleshooting#startup-failures).
    </Tip>
  </Step>

  <Step title="Send your first message">
    Reply to the startup DM to start a conversation — responses stream
    in real time. Try one of these to see the bot in action:

    > Remind me to take out the trash tomorrow at 10am, and follow up if I haven't done it by noon

    This demonstrates smart follow-ups — the bot will check back if you
    haven't confirmed the task.

    > Set up a morning routine that gives me a summary of my day every weekday at 9am

    This sets up a proactive check-in — the feature that makes ollim-bot
    different from a regular chatbot.

    You can also try `/cost` to see token usage or `/model sonnet` to
    switch models.

    <Note>
    After [setting up Google integration](/getting-started/setup), try: "What's on my calendar today?"
    </Note>
  </Step>
</Steps>

## CLI commands

Beyond running the bot, `ollim-bot` provides commands for managing
routines, reminders, and Google services. Run these in your terminal:

```bash
ollim-bot routine list     # Show all routines
ollim-bot reminder add --delay 30 -m "take a break"
ollim-bot tasks list       # List Google Tasks
ollim-bot cal today        # Show today's calendar events
ollim-bot help             # Show all available commands
```

## Troubleshooting

<AccordionGroup>
  <Accordion title='"Private application cannot have a default authorization link"'>
    This error appears on the **Installation** page when **Public Bot** is
    already unchecked. Discord does not allow private applications to have a
    default authorization link.

    **Fix:** go to **Bot** in the sidebar, check **Public Bot**, then return to
    **Installation** and configure the install link. After you've installed the
    app to your account, go back and uncheck **Public Bot**.
  </Accordion>

  <Accordion title="Bot not responding to DMs">
    Verify that **Message Content Intent** is enabled under **Bot** >
    **Privileged Gateway Intents**. Without it, the bot connects but cannot
    read your messages.
  </Accordion>

  <Accordion title="Slash commands not appearing">
    Slash commands sync on startup. If they don't appear:

    1. Confirm you installed via **User Install** (not Guild Install) — slash
       commands in DMs require user installation
    2. Wait a few minutes — Discord caches command lists and may take time to
       update
    3. Restart Discord (Ctrl+R on desktop) to force a cache refresh
  </Accordion>
</AccordionGroup>

For runtime issues (startup failures, session problems, scheduling), see
[troubleshooting](/development/troubleshooting).
