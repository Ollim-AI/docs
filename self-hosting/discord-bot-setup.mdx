This tutorial walks through creating a Discord application,
generating a bot token, and enabling the required privileged intent.
ollim-bot runs entirely in DMs — no server invite is needed.

### What you'll have after this

| Key | Value |
|-----|-------|
| **Environment variables** | `DISCORD_TOKEN`, `OLLIM_USER_NAME`, `OLLIM_BOT_NAME` in `.env` |
| **Required intent** | Message Content Intent (privileged) |
| **Installation context** | User Install (not Guild Install) |
| **Install scope** | `applications.commands` |
| **Bot setting** | Public Bot unchecked |
| **PID file** | `~/.ollim-bot/state/bot.pid` (single-instance guard) |

## Prerequisites

- A [Discord account](https://discord.com)

<Steps>
  <Step title="Create a Discord application">
    Go to the [Discord Developer Portal](https://discord.com/developers/applications)
    and click **New Application**. Give it a name (e.g. `ollim-bot`)
    and click **Create**.

    <Frame>
      <img src="/images/discord-new-app.png" alt="Discord Developer Portal showing the New Application button" />
    </Frame>

    Note the **Application ID** on the General Information page.
  </Step>

  <Step title="Create a bot and copy the token">
    In the left sidebar, click **Bot**. Click **Reset Token** (or **Add Bot** if prompted), then copy the token.

    <Frame>
      <img src="/images/discord-bot-token.png" alt="Discord Bot page showing the Reset Token button and Public Bot toggle" />
    </Frame>

    Add it to your `.env` file:

    ```bash title=".env"
    DISCORD_TOKEN=MTE5...your-token
    ```

    <Warning>The token is shown only once. If you lose it, you must reset it to generate a new one.</Warning>
  </Step>

  <Step title="Enable the Message Content intent">
    On the same **Bot** page, scroll down to **Privileged Gateway Intents**. Enable **Message Content Intent**.

    <Frame>
      <img src="/images/discord-message-intent.png" alt="Privileged Gateway Intents section with Message Content Intent enabled" />
    </Frame>

    ollim-bot requires this intent to read message content in DMs.
    The bot initialization enables it explicitly:

    ```python
    intents = discord.Intents.default()
    intents.message_content = True
    ```

    Without this intent enabled, the bot cannot see what you type.
  </Step>

  <Step title="Configure user installation">
    In the left sidebar, click **Installation**.

    1. Under **Installation Contexts**, check **User Install** (uncheck **Guild Install** — ollim-bot doesn't use servers)
    2. Set the install link to **Discord Provided Link**
    3. Under **Default Install Settings** for the user install context, add the `applications.commands` scope

    <Frame>
      <img src="/images/discord-installation.png" alt="Discord Installation page with User Install checked, Discord Provided Link selected, and applications.commands scope added" />
    </Frame>

    This lets you install the app directly to your Discord account so slash commands work in DMs without a shared server.
  </Step>

  <Step title="Install the app and disable public access">
    Copy the **Install Link** shown on the Installation page and open it in your browser. Select **Add to my apps** to install the bot to your Discord account.

    Now go back to **Bot** in the sidebar and uncheck **Public Bot** — ollim-bot is single-user, so no one else needs to install it. This must be done *after* setting the install link, because Discord doesn't allow private apps to have a default authorization link.

    <Tip>
    Bookmark the install link — you'll need it again if you re-create the application.
    </Tip>
  </Step>

  <Step title="Configure remaining environment variables">
    In addition to `DISCORD_TOKEN`, ollim-bot requires two more environment variables:

    ```bash title=".env"
    DISCORD_TOKEN=MTE5...your-token
    OLLIM_USER_NAME=YourName
    OLLIM_BOT_NAME=ollim
    ```

    | Variable | Required | Description |
    |----------|----------|-------------|
    | `DISCORD_TOKEN` | Yes | Bot token from step 2 |
    | `OLLIM_USER_NAME` | Yes | Your display name |
    | `OLLIM_BOT_NAME` | Yes | The bot's name |

    If either `OLLIM_USER_NAME` or `OLLIM_BOT_NAME` is missing, the bot exits immediately with an error.
  </Step>

  <Step title="Start the bot">
    ```bash
    ollim-bot
    ```

    Required environment variables are validated at import time (before the bot starts).
    Once running, the bot:
    1. Checks that no other instance is running (via PID file at `~/.ollim-bot/state/bot.pid`)
    2. Connects to Discord and syncs slash commands
    3. Sends you a startup DM
    4. Starts the scheduler and webhook server (if configured)

    You should see output like:

    ```text
    ollim online as ollim#1234
    synced 10 slash commands
    ```
  </Step>
</Steps>

<Tip>Send a DM to your bot to verify it responds.</Tip>

## Slash commands

Once the bot is online, these slash commands are available:

| Command | Description |
|---------|-------------|
| `/clear` | Reset conversation history |
| `/compact` | Compress conversation context |
| `/cost` | Show token usage for the session |
| `/model` | Switch model (opus, sonnet, haiku) |
| `/thinking` | Set thinking mode (off, adaptive, budget) |
| `/fork` | Start a forked conversation |
| `/interrupt` | Stop the current response |
| `/permissions` | Set permission mode |
| `/ping-budget` | View or configure ping budget |
| `/config` | View or set runtime configuration |

## Troubleshooting

<AccordionGroup>
  <Accordion title='"Private application cannot have a default authorization link"'>
    This error appears on the **Installation** page when **Public Bot** is
    unchecked. Discord does not allow private applications to have a default
    authorization link.

    **Fix:** go to **Bot** in the sidebar, check **Public Bot**, then return to
    **Installation** and configure the install link. After you've installed the
    app to your account, go back and uncheck **Public Bot**.
  </Accordion>

  <Accordion title="Bot not responding to DMs">
    Verify that **Message Content Intent** is enabled under **Bot** >
    **Privileged Gateway Intents**. Without it, the bot connects but cannot
    read your messages. See
    [step 3](#enable-the-message-content-intent) above.
  </Accordion>

  <Accordion title="Slash commands not appearing">
    Slash commands sync on startup. If they don't appear:

    1. Confirm you installed via **User Install** (not Guild Install) — slash
       commands in DMs require user installation
    2. Wait a few minutes — Discord caches command lists and may take time to
       update
    3. Restart Discord (Ctrl+R on desktop) to force a cache refresh
  </Accordion>
</AccordionGroup>

For runtime issues (startup failures, session problems, scheduling), see the
[troubleshooting guide](/development/troubleshooting).
