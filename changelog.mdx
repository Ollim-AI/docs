ollim-bot is developed continuously without formal version releases.
This changelog tracks major features and milestones in reverse
chronological order.

<Note>ollim-bot has been in active development since February 9, 2026 with 588 commits across 35 active development days.</Note>

## Mar 29-31, 2026

**Docker deployment**

- **Dockerfile and compose stack** — the repo now ships `Dockerfile`, `docker-compose.yml`, and `docker-entrypoint.sh`. `docker compose up -d` brings up ollim-bot alongside a local [Ollama](https://ollama.com) server, with named volumes (`ollama_data`, `bot_data`, `cli_data`) for persistence across rebuilds. See [Docker deployment](/self-hosting/guide#docker-deployment).
- **Auto-pull Ollama models** — the entrypoint checks whether `ANTHROPIC_MODEL` exists on the Ollama server and pulls it automatically if not, so the first start downloads the model before the bot launches.
- **Skip OAuth when `ANTHROPIC_AUTH_TOKEN` is set** — `is_authenticated()` is no longer called when the token is present, fixing a crash in restricted container environments where the bundled CLI can't write temp files.
- **Non-root container user** — the bot runs as an unprivileged `bot` user inside the container. Ollama's port is bound to `127.0.0.1` on the host, not exposed to the network.
- Internal: `--no-dev` in the Dockerfile `CMD` skips dev dependency downloads, session ID reset in chat mode prevents stale state across container restarts.

## Mar 27, 2026

**AI training restriction and source-file headers**

- **AI training restriction** — [`LICENSE.md`](https://github.com/Ollim-AI/ollim-bot/blob/main/LICENSE.md) gained a Section 7 supplementary notice prohibiting use of the repo's contents for training or evaluating machine learning models without written permission. GPL-3.0 itself was adopted on Feb 26. See [License](/self-hosting/guide#license).
- **Per-file copyright headers** — all Python source files now carry a GPL-3.0 copyright header naming Julius Frost as the copyright holder (2025-2026).
- **Crawler opt-out signals** — `robots.txt` and `ai.txt` added at the repo root to signal the restriction to automated crawlers.

## Mar 23, 2026

**`send_file` tool and save-context confirmation**

- **`send_file` MCP tool** — new [Discord tool](/extending/mcp-tools#send_file) for sending local files as Discord DM attachments. Validates paths via `path.is_file()`, enforces Discord's 25 MB limit, supports tilde expansion, and respects [ping budget](/scheduling/ping-budget) in background forks.
- **`save_context` confirmation embed** — calling `save_context` now sends a yellow confirmation embed with **Confirm**, **Report Instead**, and **Dismiss** buttons instead of promoting the fork immediately. Prevents unintended saves when the agent calls the tool proactively. The **Save Context** button on the fork embed still saves directly.

## Mar 20-22, 2026

**Onboarding and setup improvements**

- **Welcome button** — the first-run startup DM now includes a **"Get started — 3 questions"** button that triggers the [setup](/getting-started/quickstart#run-first-time-setup) skill, replacing the previous open-ended welcome message.
- **Setup creates morning check-in** — setup now offers to create a weekday morning check-in [routine](/scheduling/routines) and writes the routine file directly when the user accepts, instead of just suggesting it.
- **Google integration awareness in setup** — after the routine offer, setup checks Google integration status: skips the mention if already connected, suggests `/google-auth` if credentials exist but aren't authorized, or links to the [setup guide](/getting-started/google-integration) if nothing is configured.
- **Google status in background preamble** — [background forks](/scheduling/background-forks) now receive a Google status line in their preamble, so routines skip Google commands gracefully when not configured instead of failing silently.

## Mar 21, 2026

**Agent core changes — model auto-upgrade, bug fixes, and SDK upgrade**

- **Haiku-to-sonnet auto-upgrade** — when context exceeds haiku's 200k limit at fork creation time, the bot automatically upgrades the [interactive fork](/core-usage/forks#haiku-auto-upgrade) to sonnet. The fork embed shows a warning, and `enter_interactive_fork()` returns a `bool` indicating the upgrade. The threshold is defined in `_CONTEXT_WINDOWS` and checked against `_last_input_tokens` from `ResultMessage.usage`.
- **`claude-agent-sdk` 0.1.47 to 0.1.50** — dependency bumped in `pyproject.toml`. See the [SDK changelog](https://github.com/anthropics/claude-agent-sdk/releases) for details.
- **Model-aware context warnings** — the 60% context usage annotation now uses per-model context windows (`_CONTEXT_WINDOWS` dict) instead of a hardcoded 200k default. Opus and Sonnet correctly report against their 1M window.
- **Compaction retry guard** — compaction retry after auto-compaction is now skipped when a fork interrupt occurred (`not fork_interrupted`), preventing a wasted API call.
- **`timestamp()` timezone fix** — `timestamp()` in `agent_context.py` now uses the `%Z` format directive to reflect the configured timezone instead of hardcoding "PT".
- **Images preserved on compaction retry** — the compaction retry query now replicates the image-aware `build_image_query()` call, preventing images from being silently dropped.
- **`set_thinking()` permission restore** — `set_thinking()` now restores `_dont_ask` from persisted config when tearing down a fork client, matching the existing fork-exit behavior.
- **Fallback text after compaction** — the `streamed` flag is now reset after compaction, so fallback `ResultMessage` text is no longer silently dropped.

## Mar 20-21, 2026

**ADHD behavior eval system and developer tooling**

- **`ollim-bot eval` CLI** — new subcommand with `run`, `list`, `results`, and
  `compare` sub-subcommands for running ADHD behavior evals against the real
  agent. `eval run` plays a Haiku-powered user-proxy against the bot, then
  scores transcripts with a Sonnet-powered LLM judge. Results persist as JSONL
  for regression tracking. See [CLI reference](/development/cli-reference#eval).
- **6 ADHD scenarios** — `overwhelmed-by-tasks`, `vague-routine-request`,
  `proactive-morning-checkin`, `frustrated-nothing-works`,
  `reminder-negotiation`, and `context-recovery-after-gap`. Each defines a
  persona, goal, and ADHD-specific scoring criteria.
- **UX-grounded judge** — the LLM judge scores against ADHD behavioral
  anchors (one-thing focus, warm tone, no option dumps, context-awareness)
  on a 1-5 scale.
- **`eval compare`** — compares latest vs previous results per scenario,
  flagging regressions when overall score drops by more than 0.3.
- **33 ADHD behavior regression tests** — 3 modules (441 lines) protecting
  prompt content, multi-step interaction flows, and context assembly. These
  run in the regular `pytest` suite without API calls.
- **Proxy prompt and goal truncation fix** — strengthened the Haiku
  user-proxy prompt for more realistic ADHD behavior, fixed qualitative goal
  truncation in `_print_scores`.
- **3 SDK docs preload skills** — developer-facing Claude Code skills
  (`/sdk-docs-ref`, `/sdk-docs-core`, `/sdk-docs-full`) with tiered context
  depth for working with Claude Agent SDK source code.

## Mar 20, 2026

**Webhook skills, subagent binding, and fire-time validation**

- **Webhook skills support** — webhooks now accept a `skills` array in YAML frontmatter, matching [routines](/scheduling/routines) and [reminders](/scheduling/reminders). Referenced skill instructions and tools are loaded at fire time. Skill tool merging moved from `scheduler.py` to `BgForkConfig.with_skill_tools()` for consistent ownership across all job types.
- **Subagent binding** — routines, reminders, and webhooks support a new `subagent` field. When set, the generated skill includes a `REQUIRED SUBAGENT` delegation instruction, and the `Agent(<name> *)` tool pattern is merged into `allowed_tools` via `BgForkConfig.with_subagent_tools()`. Unknown subagents skip execution at fire time.
- **Skill reference validation** — `validate_skill_refs()` in `skills.py` checks that every skill in a job's `skills` array has a `SKILL.md` in the skills directory. Missing refs block execution entirely — routines and reminders append a pending update with the error; webhooks log a warning and skip dispatch.
- **Webhook screening timeout** — injection screening now enforces a 30-second timeout (`_SCREENING_TIMEOUT`) via `asyncio.timeout`. If screening times out or fails, the webhook proceeds (fail-open).
- **User skills in system prompt** — `list_user_skills()` enumerates skills that are neither generated (`routine-*`, `reminder-*`, `webhook-*`) nor bundled, and injects a dynamic "Your custom skills" section into the system prompt with each skill's name and description. Inline `SKILL.md` format instructions replace the previous "search the docs" reference.

## Mar 17-18, 2026

**Guide subagent renamed to ollim-bot-guide**

- **`guide` renamed to `ollim-bot-guide`** — the subagent spec file moved from `subagents/guide.md` to `subagents/ollim-bot-guide.md`, and all references in prompts and tool policy updated to match.
- **Improved prompt** — the rewritten prompt adds explicit source priority tiers (docs, then user config, then CLI, then flagged inference), a docs discovery step via the `llms.txt` index, a richer uncertainty protocol (covering ambiguous docs, conflicting sources, and wrong assumptions in questions), proactive related feature suggestions, and graceful scope boundary handling that redirects runtime debugging questions.
- **Better trigger description** — the subagent description now addresses the decision-maker directly, enumerates domain categories, and includes a self-use trigger for when the main agent itself needs docs knowledge.

## Mar 17-18, 2026

**Tool error display, bug fixes, and code quality**

- **Tool error labels** — tools that fail during execution now show with
  strikethrough and an " — error" suffix in the stream
  (`~~Tool(args)~~ — error`), matching the existing denied-tool styling.
  Interrupted tool failures are skipped. See
  [Visual feedback](/core-usage/permissions#visual-feedback).
- **Async event loop fixes** — synchronous I/O calls (file writes,
  JSONL appends, subprocess invocations) wrapped in
  `asyncio.to_thread()` to prevent blocking the event loop.
- **Background fork model forwarding** — `model` parameter now forwarded
  to `create_forked_client()` in non-isolated background forks, so
  per-job model overrides take effect.
- **Cron field validation** — `IndexError` from malformed cron fields
  handled in the scheduler and preamble instead of crashing.
- **Storage serialization fixes** — `_yaml_escape()` now escapes quotes
  and backslashes, `isinstance` type comparison corrected, and corrupt
  JSONL lines are logged and skipped instead of crashing.
- **Embed fallback defaults** — invalid `color` or `style` values from
  the agent now fall back to defaults instead of raising errors.
- **`format_error` type handling** — `format_error()` in `updater.py`
  now handles both `str` and `bytes` stderr to prevent `AttributeError`
  when `.decode()` was called on a string.
- **Redundant status suppression** — duplicate status messages suppressed
  once a text message already exists in the stream.
- **`save_context` gating** — `save_context` now requires an explicit
  user request and is no longer called proactively by the agent.
- Internal: `_check_and_consume` helper extracted in `permissions.py` to
  eliminate duplication between denied and errored label checks.
- Internal: python-principles audit trimmed redundant docstrings across
  20+ files.

## Mar 16, 2026

**Diagnostic CLI and test coverage**

- **`ollim-bot doctor`** — new diagnostic command that checks every layer of
  the routine execution pipeline: environment variables, data directory,
  timezone, routines, reminders, tool policy, state files, Claude CLI, and
  authentication. Reports **PASS**/**WARN**/**FAIL** with actionable messages.
  See [CLI reference](/development/cli-reference) and
  [Troubleshooting](/development/troubleshooting).
- **`tasks show`** — view full task details (title, status, due date, notes) for any task. The task list also now shows a **[+]** marker when notes exist. See [Google Tasks](/integrations/google-tasks).
- **Scheduler self-notification** — corrupt routine/reminder files and failed
  cron registrations now surface to the agent via pending updates, enabling
  self-correction on the next interaction.
- **Defensive file I/O** — `safe_json_load()` utility and explicit UTF-8
  encoding on all data file operations.
- Internal: 1,371 lines of new tests across 5 modules — fork state, agent
  state transitions, agent context, reminder tools, and system prompts
  (135 total test cases added).

## Mar 12-15, 2026

**Reminder MCP tools, fork resilience, and platform fixes**

- **Reminder MCP tools** — reminders are now managed through
  `add_reminder`, `list_reminders`, and `cancel_reminder`
  [MCP tools](/extending/mcp-tools) instead of CLI commands. The
  `add_reminder` tool accepts `delay_minutes` or `run_at` for scheduling,
  plus a `foreground` flag for the rare case where you want to watch tool
  execution stream live. Reminders remain background by default.
- **Reply-to-fork resume** — replying to an **interactive fork** message
  now correctly resumes that fork session, matching the existing behavior
  for [background forks](/scheduling/background-forks).
- **Staleness notice on fork resume** — when resuming a
  **background fork** as interactive (via reply or button), the agent sees
  a notice that the **main session** has diverged. `save_context` is
  hard-blocked for resumed forks to prevent silent history loss.
- **Fork exit guard** — `report_updates`, `save_context`, and `exit_fork`
  now require at least one user turn before they can be called, preventing
  premature exits on fork entry.
- **Stale-fork save protection** — `save_context` is blocked when a fork
  is stale or pending updates exist, avoiding silent overwrites of
  **main session** history.
- **Discord 503 retry** — transient Discord API outages now retry up to
  5 times with exponential backoff (5s to 80s) instead of crashing.
- **Progressive tool labels** — tool activity labels render incrementally
  during streaming instead of waiting for the tool call to finish.
- **CLI 2.1.x compatibility** — handles the `Task` to `Agent` tool rename
  in Claude CLI 2.1.x.
- **Slash command current values** — `/permissions`, `/model`, and
  `/thinking` now display their current value when called without
  arguments.
- **Windows fixes** — `sys.argv` used directly for restart (fixes
  path-quoting issues) and UTF-8 encoding forced on the Windows console
  for all CLI commands.
- Internal: consolidated `stream_to_channel` calls through `_dispatch`,
  extracted `GATED_TOOLS` constant for **background fork** tool stripping,
  removed redundant `uv sync` from auto-updater.

## Mar 11, 2026

**Job-config skill, safety gating, and fork exit controls**

- **Job-config skill** — new bundled [skill](/extending/skills) that guides
  tool selection and behavioral configuration when creating background jobs
  (routines, reminders, webhooks). The agent is instructed to invoke it when creating or editing
  background jobs.
- **Discord tools stripped from allowed-tools** — Discord tools
  (`ping_user`, `discord_embed`, `report_updates`, `follow_up_chain`) are now
  automatically removed from `allowed-tools` in
  [background forks](/scheduling/background-forks). They're gated by
  `allow-ping` and `update-main-session` flags instead, preventing bypass
  of safety gating.
- **Fork exit tools require explicit request** — in interactive forks, the
  bot no longer calls `report_updates`, `save_context`, or `exit_fork` on
  its own. It waits for you to explicitly ask to wrap up or leave.
  See [Forks](/core-usage/forks).
- **Background fork permission hardening** —
  [background forks](/scheduling/background-forks) now always use
  `permission_mode="default"` regardless of the main session's setting.
  This prevents `bypassPermissions` from leaking into background forks,
  ensuring tool gating and [ping budget](/scheduling/ping-budget)
  enforcement are never skipped.
- Internal: `--foreground`/`--background` mutual exclusivity in reminder
  CLI, isolated reminder label format updated to `[bg,isolated]`.
- Internal: switched from a custom
  [system prompt](/development/system-prompt) string to the `claude_code`
  preset with appended ollim-bot instructions — preserves Claude Code's
  built-in tool usage guidelines and safety instructions.

## Mar 10-11, 2026

**Reminders default to background, fork prompt improvements**

- **Reminders default to background** — reminders now run as
  [background forks](/scheduling/background-forks) by default, letting the
  agent decide at fire-time whether to ping. The `--foreground` flag
  explicitly opts into direct-DM mode. The old `--background` flag is
  deprecated (still accepted, prints a warning).
- **Fork prompt improvements** — interactive fork topic prompts reframed
  from "respond then consider exit" to "engage and follow the user's lead."
  Pre-loaded timeout and exit-strategy guidance removed from the system
  prompt — exit rules are now delivered just-in-time by `[fork-timeout]`
  prompts and scoped to agent-initiated forks only.
- **Terminology consistency** — docstrings and internal prompts updated
  to use "background fork" consistently (docs already used correct terms).
- **Context usage warning** — a small annotation appears after responses
  when context exceeds 60% of the 200k window
  (`context: 67% (134k) — consider /compact`). Escalates at 80% with
  "compaction soon" messaging, giving you a heads-up to
  [compact proactively](/core-usage/conversations#context-and-memory)
  before auto-compaction kicks in.
- **/setup wizard** — guided `/setup` command that walks through
  configuring names, [personality](/personalizing/personality), and
  [user context](/personalizing/user-context). Detects fresh vs
  configured state and offers fast or extended paths.
- **Nested tool activity** — subagent status messages now show what tool
  the subagent is using (e.g., `Task(desc) · Read... (12s)`) instead of
  a static timer.
- Internal: extracted shared formatting helpers, deferred task progress
  Discord edits for rate limit relief, updated history-reviewer for
  current `claude-history` CLI, made setup skill self-contained.

## Mar 9-10, 2026

**Profile files, attachment downloads, and fork permission scoping**

- **Profile files** — bot personality extracted from hardcoded prompts into
  user-editable [`IDENTITY.md`](/personalizing/personality) and
  [`USER.md`](/personalizing/user-context) files in `~/.ollim-bot/`.
  `IDENTITY.md` is bootstrapped on first run with a default persona;
  `USER.md` provides user context. Both are prepended to the system prompt
  and editable by the agent at runtime.
- **Attachment downloads** — non-image Discord attachments (PDFs, text files,
  etc.) are now saved to `~/.ollim-bot/downloads/` so the agent can read
  them. Images continue through base64 encoding as before.
- **/config options display** — each config key now shows its valid options
  next to the current value, making it easier to see what you can change.
- **Fork permission scoping** — permission mode changes during a
  [fork](/core-usage/forks) now stay fork-scoped. Exiting a fork restores
  the persisted `_dont_ask` setting, and stale denied tool labels are
  cleared at each new response.
- **Gender-neutral language** — system prompt and profile module updated to
  use they/them pronouns throughout.

## Feb 28, 2026

**User-proxy subagent and shutdown notifications**

- **User-proxy subagent** — a fourth subagent (`haiku`-powered, read-only)
  that answers "what would the user do?" questions during background forks.
  Searches preference files and conversation history, returns answers with
  calibrated confidence (HIGH/MEDIUM/LOW). Background fork preambles
  conditionally include a hint to use it when the `Task` tool is available.
- **Shutdown DM** — the bot now sends a DM to the owner before shutting
  down on crash, SIGINT, or SIGTERM. The message is best-effort and
  includes the shutdown reason (e.g. `shutting down: received SIGTERM`).
